# Mesh Elevations

## 1. Overview
Elevations define the vertical layers of the mesh.  
Each elevation represents a distinct operational altitude.

The mesh is not flat — it is stratified.

## 2. Elevation Levels

### Elevation 0 — Ground State
- Local identity  
- Direct hardware interaction  
- ENU coordinate anchoring  
- No recursion

### Elevation 1 — Chamber State
- Chamber JSON modes (bind, banish, entity, invoke)  
- Ritual engine integration  
- Localized sovereignty

### Elevation 2 — Capsule State
- Recursive identity  
- Multi‑signature governance  
- Cross‑node continuity  
- Forensic logging

### Elevation 3 — Mesh State
- Distributed routing  
- Burst propagation  
- Pressure‑based optimization  
- Autonomous recovery

### Elevation 4 — Sovereign State
- Constitutional enforcement  
- Jurisdictional boundaries  
- Multi‑branch governance  
- Emergency continuity protocols

## 3. Elevation Transitions
Transitions are triggered by:

- Pressure thresholds  
- Capsule recursion  
- Chamber invocation  
- Backbone signals  
- External routing attempts

## 4. Elevation Rules
- Lower elevations cannot override higher ones  
- Higher elevations inherit but do not depend on lower ones  
- Sovereign elevation is immutable once activated

This file defines the vertical dimension of the mesh.
